🧠 大模型 API 是什么？ 必做

大模型 API 就是一个"远程大脑服务"——你通过 HTTP 请求把问题发过去，它把回答发回来。你不需要在自己电脑上跑模型，只需要会调接口。

概念解释类比
API Key你的身份凭证，用于认证和计费像银行卡号
Model选哪个大脑（GPT-4、Claude、通义千问等）选哪个医生看病
Messages对话历史（角色 + 内容）聊天记录
Token文本的计量单位（约 1 个中文字 ≈ 1-2 个 token）计费单位，像手机流量
Temperature控制回答的"创造性"（0=严格，1=发散）像调节创意度旋钮
🔑 获取 API Key 必做

选一个平台注册，获取 API Key。推荐先从一个开始，后面再扩展。

平台模型注册地址特点

AnthropicClaude 4 系列console.anthropic.com长文本强，安全性好
阿里云通义千问dashscope.aliyun.com国内访问无障碍，有免费额度

🚀 第一次 API 调用（OpenAI 示例） 必做
TERMINAL — 安装 SDK
pip install openai python-dotenv
first_call.py — 你的第一次 AI 调用
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()

# ---- 创建客户端 ----
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# ---- 发送请求 ----
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个乐于助人的 AI 助手。"},
        {"role": "user", "content": "用一句话解释什么是机器学习"}
    ],
    temperature=0.7,
    max_tokens=200
)

# ---- 提取回答 ----
answer = response.choices[0].message.content
tokens_used = response.usage.total_tokens

print(f"AI 回答: {answer}")
print(f"消耗 token: {tokens_used}")

📌 messages 的三种角色

  • system — 设定 AI 的身份和行为规则（"你是…"）
  • user — 用户说的话
  • assistant — AI 之前的回答（多轮对话时需要）
🔄 用 requests 原生调用（理解底层原理） 重要

SDK 是对 HTTP 请求的封装。理解底层原理后，你可以对接任何大模型 API——它们格式几乎一样。

raw_request.py — 用 requests 直接调用
import requests
import os
from dotenv import load_dotenv

load_dotenv()

url = "https://api.openai.com/v1/chat/completions"
headers = {
    "Authorization": f"Bearer {os.getenv('OPENAI_API_KEY')}",
    "Content-Type": "application/json"
}
payload = {
    "model": "gpt-4o",
    "messages": [
        {"role": "user", "content": "什么是 Prompt Engineering？"}
    ],
    "temperature": 0.7
}

response = requests.post(url, headers=headers, json=payload)
data = response.json()

print(data["choices"][0]["message"]["content"])

💡 国内平台通常只需要换 url 和 api_key

大多数国内大模型（DeepSeek、通义千问、智谱等）都兼容 OpenAI 格式。只需改 base_url 和 api_key，代码几乎不用动。

DAY3-5
Prompt Engineering 核心技巧
AI 的表现 80% 取决于你怎么提问——这是最关键的技能
🎯 技巧 1：明确角色（System Prompt） 必做

System Prompt 是给 AI 的"工作手册"——告诉它是谁、怎么做、有什么规矩。好的 System Prompt 能让 AI 表现提升数倍。

差的 System Prompt： 你是一个助手。
好的 System Prompt： 你是一位资深 Python 开发者，专注于数据处理和 AI 应用开发。

你的工作方式：
- 回答简洁直接，先给代码，再给解释
- 代码中加注释说明关键步骤
- 如果问题不明确，先列出你的假设再回答
- 涉及安全问题时（如 API Key），必须提醒最佳实践

限制：
- 不要给出已过时的库用法（如 Python 2 语法）
- 不确定的信息要明确说"我不确定"
system_prompt.py
system_prompt = """你是一位资深 Python 开发者，专注于数据处理和 AI 应用开发。

你的工作方式：
- 回答简洁直接，先给代码，再给解释
- 代码中加注释说明关键步骤
- 如果问题不明确，先列出你的假设再回答

限制：
- 不确定的信息要明确说"我不确定"
"""

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": "怎么用 Pandas 读 CSV 并去重？"}
    ]
)

给 AI 看几个"例子"，它就知道你要什么格式和风格。这是最有效的提示技巧之一。

messages = [

    # ---- 示例 1（Few-shot）----
    {"role": "user", "content": "这家餐厅味道很好，服务也棒！"},

    # ---- 示例 2 ----

    {"role": "assistant", "content": '{"sentiment": "negative", "confidence": 0.92, "keywords": ["等了一个小时", "差评"]}'},

    # ---- 真正要处理的输入 ----

    temperature=0   # 结构化输出建议 temperature=0
)
print(response.choices[0].message.content)
# {"sentiment": "mixed", "confidence": 0.85, "keywords": ["价格还行", "环境吵"]}
🔗 技巧 3：思维链（Chain of Thought） 必做

让 AI "一步步思考"再给结论，能显著提升推理准确率。对数学、逻辑、复杂分析类任务特别有效。

差的方式（直接要答案）： 小明有 5 个苹果，给了小红 2 个，又买了 3 个，小红还了 1 个，现在小明有几个？
好的方式（引导思维链）： 小明有 5 个苹果，给了小红 2 个，又买了 3 个，小红还了 1 个，现在小明有几个？

请一步步思考，先列出每一步的变化，最后给出答案。

📌 Prompt 工程核心原则总结

  • 明确角色 — 告诉 AI 它是谁、怎么做
  • 给示例 — Few-shot 比纯描述有效 10 倍

  • 限定格式 — 要 JSON 就写清楚 JSON 结构
  • 减少歧义 — 越具体越好，别让 AI 猜

📋 Prompt 模板化（工程化的关键） 必做

实际工作中，Prompt 不是写死的字符串，而是可复用的"模板"——用变量动态填充内容。

prompt_template.py
# ---- 基础模板 ----
SUMMARY_TEMPLATE = """请对以下文章进行摘要。

要求：
- 语言：{language}
- 长度：{length}字以内
- 风格：{style}

文章内容：
---
{article}
---

请输出摘要："""

# ---- 动态填充 ----
prompt = SUMMARY_TEMPLATE.format(
    language="中文",
    length=100,
    style="专业简洁",
    article="今天 OpenAI 发布了新版模型..."
)

# ---- 更优雅的方式：用字典管理多个模板 ----
TEMPLATES = {
    "summarize": "请用{length}字总结：\n{text}",
    "translate": "将以下内容翻译成{lang}：\n{text}",
    "classify":  "将以下文本分类为{categories}之一：\n{text}",
    "extract":   "从以下文本中提取{fields}：\n{text}",
}

def build_prompt(task, **kwargs):
    template = TEMPLATES.get(task)
    if not template:
        raise ValueError(f"未知任务: {task}")
    return template.format(**kwargs)

# 使用
prompt = build_prompt("translate", lang="英文", text="今天天气真好")
print(prompt)  # 将以下内容翻译成英文：\n今天天气真好
🔄 批量处理 + Pandas 联动 必做

真实工作中，你经常需要对一批数据逐条调用 AI。这就是 Phase 1-2 学的 Pandas 大显身手的时候。

batch_processing.py
import pandas as pd
import time
from openai import OpenAI

client = OpenAI()

# ---- 读取待处理数据 ----
df = pd.read_csv("reviews.csv")  # 假设有 text 列

def analyze_sentiment(text):
    """调用 AI 分析情感"""
    try:
        response = client.chat.completions.create(
            model="gpt-4o-mini",  # 批量处理用便宜的模型
            messages=[
                {"role": "system", "content": "分析情感，只输出: positive/negative/neutral"},
                {"role": "user", "content": text}
            ],
            temperature=0,
            max_tokens=10
        )
        return response.choices[0].message.content.strip()
    except Exception as e:
        print(f"Error: {e}")
        return "error"

# ---- 批量处理（加延迟避免限速）----
results = []
for i, text in enumerate(df["text"]):
    sentiment = analyze_sentiment(text)
    results.append(sentiment)
    print(f"{i+1}/{len(df)} 完成: {sentiment}")
    time.sleep(0.5)  # 防止 API 限速

# ---- 结果写回 DataFrame ----
df["sentiment"] = results
df.to_csv("reviews_analyzed.csv", index=False)
print("分析完成！")
print(df["sentiment"].value_counts())

🏗️ Structured Outputs（模式约束） 重要

比 JSON Mode 更严格——你可以定义一个 JSON Schema，AI 的输出会 100% 符合这个结构。

structured_output.py
from pydantic import BaseModel
from typing import List

# ---- 用 Pydantic 定义输出结构 ----
class ReviewAnalysis(BaseModel):
    sentiment: str       # positive / negative / neutral
    score: int           # 1-10
    summary: str         # 一句话总结
    tags: List[str]      # 标签列表

# ---- 使用 parse 方法（OpenAI SDK 支持）----
completion = client.beta.chat.completions.parse(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "分析产品评论"},
        {"role": "user", "content": "手机拍照很好但发热严重"}
    ],
    response_format=ReviewAnalysis
)

result = completion.choices[0].message.parsed
print(result.sentiment)   # 直接当 Python 对象用！
print(result.score)
print(result.tags)

💡 三种获取结构化输出的方式（从简单到严格）

① Prompt 中描述格式 → 最简单，但 AI 可能不遵守
② JSON Mode → 保证输出是合法 JSON，但字段不确定
③ Structured Outputs → 100% 符合你定义的 Schema，最可靠

💬 多轮对话的工作原理 必做

大模型本身是"无记忆"的！每次调用都是独立的。要让 AI "记住"之前的对话，你必须把完整的对话历史每次都发过去。

multi_turn.py
from openai import OpenAI

client = OpenAI()

# ---- 维护对话历史 ----
conversation = [
    {"role": "system", "content": "你是一个友好的 AI 助手。"}
]

def chat(user_input):
    # 1. 把用户消息加入历史
    conversation.append({"role": "user", "content": user_input})

    # 2. 把完整历史发给 API
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=conversation
    )

    # 3. 把 AI 回复加入历史
    ai_msg = response.choices[0].message.content
    conversation.append({"role": "assistant", "content": ai_msg})

    return ai_msg

# ---- 模拟对话 ----
print(chat("我叫小明，我在学 Python"))
print(chat("我刚才说我叫什么？"))       # AI 能记住：小明
print(chat("给我推荐下一步学什么"))    # AI 知道你在学 Python
✂️ 上下文窗口管理（Token 限制） 必做

对话历史越长，token 消耗越大。超出模型的上下文窗口限制就会报错。需要策略来控制长度。

context_management.py
def trim_conversation(messages, max_messages=20):
    """保留 system prompt + 最近 N 轮对话"""
    system = [m for m in messages if m["role"] == "system"]
    history = [m for m in messages if m["role"] != "system"]
    return system + history[-max_messages:]

def summarize_history(messages, client):
    """当历史过长时，让 AI 总结之前的对话"""
    old_history = messages[1:-4]  # 除了 system 和最近 2 轮
    summary_prompt = "请用 3 句话总结以下对话的要点：\n"
    for m in old_history:
        summary_prompt += f"{m['role']}: {m['content']}\n"

    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": summary_prompt}],
        max_tokens=200
    )
    summary = resp.choices[0].message.content

    # 用摘要替换旧历史
    return [
        messages[0],  # system prompt
        {"role": "system", "content": f"之前的对话摘要：{summary}"},
        *messages[-4:]  # 最近 2 轮
    ]

🔧 Function Calling 是什么？ 必做

AI 不能直接查天气、读数据库、发邮件。但你可以告诉 AI"你有这些工具可以用"，AI 会判断什么时候需要调用，并返回调用参数，你的代码执行后把结果告诉 AI，AI 再整合成最终回答。

function_calling.py
import json
from openai import OpenAI

client = OpenAI()

# ---- Step 1: 定义工具（函数描述）----
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称，如：北京"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

# ---- Step 2: 你的实际函数 ----
def get_weather(city):
    # 实际项目中这里调用天气 API
    weather_data = {
        "北京": "晴，25°C",
        "上海": "多云，22°C",
    }
    return weather_data.get(city, f"暂无{city}的天气数据")

# ---- Step 3: 发送请求 ----
messages = [{"role": "user", "content": "北京今天天气怎么样？"}]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    tools=tools
)

# ---- Step 4: 检查 AI 是否要调用工具 ----
msg = response.choices[0].message

if msg.tool_calls:
    # AI 决定调用工具
    tool_call = msg.tool_calls[0]
    func_name = tool_call.function.name
    func_args = json.loads(tool_call.function.arguments)

    print(f"AI 要调用: {func_name}({func_args})")

    # 执行函数
    result = get_weather(**func_args)

    # ---- Step 5: 把结果告诉 AI ----
    messages.append(msg)  # AI 的调用请求
    messages.append({
        "role": "tool",
        "tool_call_id": tool_call.id,
        "content": result
    })

    # 再次调用，AI 会整合工具结果给出最终回答
    final = client.chat.completions.create(
        model="gpt-4o",
        messages=messages
    )
    print(final.choices[0].message.content)
    # → "北京今天天气晴朗，气温 25°C，适合外出。"

📌 Function Calling 的流程

  • ① 你告诉 AI 有哪些工具可用（tools 参数）
  • ② AI 判断是否需要工具，返回调用参数
  • ③ 你的代码执行函数，拿到结果
  • ④ 把结果喂回 AI，它生成最终回答

multi_tools.py — 通用工具调用框架
# ---- 工具注册表 ----
TOOL_REGISTRY = {
    "get_weather": get_weather,
    "search_product": search_product,
    "calculate": calculate,
}

def run_conversation(user_input, tools, max_rounds=5):
    """自动处理多轮工具调用"""
    messages = [
        {"role": "system", "content": "你是一个智能助手，可以使用工具完成任务。"},
        {"role": "user", "content": user_input}
    ]

    for _ in range(max_rounds):
        response = client.chat.completions.create(
            model="gpt-4o", messages=messages, tools=tools
        )
        msg = response.choices[0].message

        # 如果没有工具调用，就是最终回答
        if not msg.tool_calls:
            return msg.content

        # 处理所有工具调用
        messages.append(msg)
        for tc in msg.tool_calls:
            func = TOOL_REGISTRY[tc.function.name]
            args = json.loads(tc.function.arguments)
            result = func(**args)
            messages.append({
                "role": "tool",
                "tool_call_id": tc.id,
                "content": str(result)
            })

    return "达到最大调用轮数"

cost_tracker.py
class CostTracker:
    PRICING = {
        "gpt-4o": {"input": 2.5, "output": 10.0},
        "gpt-4o-mini": {"input": 0.15, "output": 0.6},
    }

    def __init__(self):
        self.total_cost = 0
        self.call_count = 0

    def track(self, response, model):
        usage = response.usage
        prices = self.PRICING.get(model, {"input": 0, "output": 0})
        cost = (usage.prompt_tokens * prices["input"]
              + usage.completion_tokens * prices["output"]) / 1_000_000
        self.total_cost += cost
        self.call_count += 1
        return cost

    def report(self):
        print(f"总调用: {self.call_count} 次, 总成本: ${self.total_cost:.4f}")

# 使用
tracker = CostTracker()
resp = client.chat.completions.create(model="gpt-4o-mini", messages=[...])
tracker.track(resp, "gpt-4o-mini")
tracker.report()
🧠 模型选型策略 必做

📌 实用选型原则

  • 简单分类/提取 → GPT-4o-mini / DeepSeek（便宜 + 够用）
  • 复杂推理/创作 → GPT-4o / Claude Sonnet（质量优先）
  • 超长文本 → Claude（200K 上下文窗口）
  • 批量处理 → 先用 mini 模型，抽检质量，不达标再换大模型
  • 缓存策略 → 相同输入缓存结果，避免重复调用
  • max_tokens → 始终设置上限，防止 AI 过度输出烧钱

def query_order(order_id):
    orders = {
        "ORD001": {"status": "已发货", "item": "无线耳机", "eta": "明天到达"},
        "ORD002": {"status": "待发货", "item": "蓝牙音箱", "eta": "预计3天"},
    }

class CustomerServiceBot:
    def __init__(self):
        self.messages = [{"role": "system", "content": SYSTEM_PROMPT}]
        self.log = []

    def chat(self, user_input):
        self.messages.append({"role": "user", "content": user_input})

        # 可能需要多轮工具调用
        for _ in range(3):
            resp = client.chat.completions.create(
                model="gpt-4o-mini",
                messages=self.messages,
                tools=tools
            )
            msg = resp.choices[0].message

            if not msg.tool_calls:
                self.messages.append(msg)
                self._log(user_input, msg.content, resp.usage.total_tokens)
                return msg.content

            # 执行工具调用
            self.messages.append(msg)
            for tc in msg.tool_calls:
                func = TOOL_MAP[tc.function.name]
                result = func(**json.loads(tc.function.arguments))
                self.messages.append({
                    "role": "tool", "tool_call_id": tc.id, "content": result
                })

    def _log(self, user_msg, ai_msg, tokens):
        self.log.append({
            "time": datetime.now().isoformat(),
            "user": user_msg, "ai": ai_msg, "tokens": tokens
        })

    def save_log(self, path="chat_log.json"):
        with open(path, "w", encoding="utf-8") as f:
            json.dump(self.log, f, ensure_ascii=False, indent=2)
